# 视频解析器桌面应用版 — 设计文档

> 日期：2026-06-24 | 状态：待实施

## 目标

将当前 Web 版视频解析器改造为独立桌面窗口应用，双击 EXE 弹出原生窗口，内嵌 WebView2 渲染界面，后台 Node.js 静默运行，不依赖外部浏览器。

## 技术方案

**C# WinForms + WebView2 + 内嵌 Node.js**

```
┌──────────────────────────────────────┐
│  WinForms 窗口 (900×650, 不可调整)     │
│  ┌────────────────────────────────┐  │
│  │  WebView2 控件                   │  │
│  │  (加载 http://localhost:3457)   │  │
│  │  ┌──────────────────────────┐  │  │
│  │  │  深色专业 UI (方案A风格)    │  │  │
│  │  │  侧栏 + 解析 + 历史 + 设置  │  │  │
│  │  └──────────────────────────┘  │  │
│  └────────────────────────────────┘  │
│  后台: node.exe server.js (无窗口)    │
└──────────────────────────────────────┘
```

| 组件 | 技术 |
|---|---|
| 窗口框架 | .NET WinForms (csc.exe 编译) |
| 浏览器内核 | WebView2 (Windows 10/11 自带) |
| 后端 | Node.js portable (内嵌) |
| 前端 | HTML/CSS/JS (重写为方案A风格) |
| 打包 | C# 资源嵌入 ZIP，首次运行解压 |

## 架构改动

### 新增文件
- `desktop/Program.cs` — 主程序入口，启动 Node + 创建 WebView2 窗口
- `frontend/` — 重写为方案 A 深色专业风格

### 修改文件
- `server.js` — 添加进程间通信，收到关闭信号时优雅退出

### 不变文件
- `backend/` — 解析引擎和路由完全复用

## 窗口规格

| 属性 | 值 |
|---|---|
| 标题 | 视频解析器 |
| 尺寸 | 1000×680 (最小) |
| 可调整 | 是 |
| 标题栏 | 自定义（模拟 macOS 风格三个圆点） |
| 关闭行为 | 同时终止 Node 进程 |
| 图标 | 系统默认 |

## 界面风格（方案 A）

- 左侧导航：解析中心 / 历史记录 / 设置
- 深色背景 (#0f1119) + 蓝色强调 (#3b82f6)
- 顶部自定义标题栏（三个圆点）
- 卡片式任务列表
- 状态标签：等待中(黄色) / 完成(绿色) / 失败(红色)

## 打包方式

- 编译 C# → `VideoParser.exe` (主程序 + WebView2 逻辑)
- 内嵌 ZIP: `node/`、`node_modules/`、`frontend/`、`backend/`、`server.js`
- 首次运行自动解压到 `%LOCALAPPDATA%\VideoParser\`
- 文件大小: ~40MB

## 启动流程

```
双击 EXE → 检查是否首次运行 → 解压文件 → 
启动 node server.js (静默) → 创建 WebView2 窗口 →
加载 http://localhost:3457 → 显示界面
```

## 前置依赖

- Windows 10 1809+ 或 Windows 11
- .NET Framework 4.7.2+ (系统自带)
- WebView2 Runtime (系统自带，未安装时需手动装一次)
